Preguntas sobre HTTP y HTML

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

❥ GET nos ayuda a solicitar alguna representación de algún recurso específico.

❥ HEAD es similar a GET en cuanto a que pide una respuesta, sin embargo, no utiliza el cuerpo de la respuesta.

❥ POST envía una entidad a un recurso específico.

❥ PUT nos ayuda a reemplazar las representaciones actuales con las deseadas.

❥ PATCH nos ayuda a aplicar modificaciones parciales a un recurso.

❥ DELETE borra algún recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML?

POST, ya que es más seguro debido a que no muestra la URL y no tiene limitaciones en la cantidad de datos enviados.

¿Qué método HTTP se utiliza para acceder a una página a través de un URL?

GET.

¿Qué significa una respuesta HTTP con código 200?

Significa que la solicitud fue exitosa y la página cargó correctamente.

¿Es responsabilidad del desarrollador corregir un error 404?

Sí, porque indica que hay un problema con el sitio web.

¿Es responsabilidad del desarrollador corregir un error 500?

Sí, porque hay un error en el servidor que debe corregirse para mejorar la experiencia del usuario.

¿Qué significa que un atributo HTML5 esté depreciado?

Significa que no se recomienda su uso. Algunas etiquetas obsoletas son: applet, acronym, bgsound, dir, hgroup, entre otras.

¿Diferencias entre HTML4 y HTML5?

HTML5 ofrece soporte para audio y video, compatibilidad con SVG, bases de datos web y una declaración de tipo de documento más corta.

¿Componentes de estructura y estilo de una tabla?

Encabezados, pies de tabla, celdas, filas y columnas.

¿Principales controles de una forma HTML5?

Botón, número, búsqueda, teléfono, URL, email, fecha, hora, color, rango, progress y meter.
